# 📝 PRD: **Memora** – AI 음성 메모장 웹서비스

---

## Overview

**Memora**는 AI 기반 웹 메모 애플리케이션으로, 사용자의 음성 또는 텍스트 입력을 받아 **AI가 자동으로 요약 및 분류**해주는 기능을 제공합니다. 

### Problem Statement
기존 메모 서비스는 사용자의 메모를 단순 저장하는 데 그치는 경우가 많아, 시간이 지날수록 중요한 정보를 찾기 어렵고 효율적인 활용이 어려워집니다. 특히 회의록, 아이디어, 공부 내용 등 다양한 형태의 메모를 빠르게 작성하고 구조화하는 것이 점점 더 중요해지고 있습니다.

### Target Users
- **직장인/프리랜서**: 회의나 작업 중 웹에서 빠르게 메모하고, 자동 정리되길 원함
- **대학생**: 노트북을 통해 수업 내용을 음성/텍스트로 기록하고 정리하길 원함  
- **일반 사용자**: 웹에서 손쉽게 일상 메모를 작성하고 필요할 때 검색하길 원함

### Value Proposition
- **AI 요약**: 긴 메모를 간결한 핵심 요약으로 제공
- **카테고리 분류**: 자동 분류 기능을 통한 체계적인 메모 정리
- **STT 기반 음성 입력**: 빠른 음성 기반 메모 작성 가능
- **웹 접근성**: 설치 없이 브라우저에서 사용 가능

## Core Features

### 1. 텍스트 메모 작성
- **What it does**: 사용자가 직접 웹에서 텍스트 입력으로 메모 작성
- **Why it's important**: 기본적인 메모 작성 기능으로 모든 사용자가 활용 가능
- **How it works**: React 기반 텍스트 에디터 컴포넌트, 실시간 저장

### 2. 음성 입력 (STT 변환)
- **What it does**: 마이크 입력을 실시간 텍스트로 변환
- **Why it's important**: 빠른 메모 작성과 접근성 향상
- **How it works**: Web Speech API 또는 외부 STT 서비스 연동

### 3. AI 요약
- **What it does**: 긴 메모 내용을 간결하게 요약
- **Why it's important**: 메모의 핵심 내용을 빠르게 파악 가능
- **How it works**: OpenAI API 또는 유사 AI 서비스 연동

### 4. 자동 카테고리 분류
- **What it does**: 메모 내용을 주제 기반으로 자동 분류
- **Why it's important**: 체계적인 메모 정리와 검색 효율성 향상
- **How it works**: AI 기반 텍스트 분류 알고리즘

### 5. 검색 기능
- **What it does**: 키워드, 작성일, 카테고리별 검색 가능
- **Why it's important**: 저장된 메모를 효율적으로 찾기 위함
- **How it works**: 데이터베이스 인덱싱과 검색 알고리즘

### 6. 데이터베이스 저장
- **What it does**: 모든 메모는 서버의 DB에 저장
- **Why it's important**: 데이터 영속성과 동기화
- **How it works**: PostgreSQL/MySQL 데이터베이스 연동

### User Personas

#### 👩‍💼 직장인 / 프리랜서
- **Pain Points**: 회의 중 빠른 메모 작성, 후속 작업을 위한 정리
- **Goals**: 효율적인 회의록 작성, 아이디어 정리
- **Tech Comfort**: 중간~높음

#### 🎓 대학생
- **Pain Points**: 수업 내용 기록, 복습을 위한 정리
- **Goals**: 체계적인 학습 노트 관리
- **Tech Comfort**: 높음

#### 👨‍👩‍👧‍👦 일반 사용자
- **Pain Points**: 일상 메모 작성, 필요시 빠른 검색
- **Goals**: 간편한 메모 작성과 관리
- **Tech Comfort**: 낮음~중간

### Key User Flows

#### 시나리오: 웹에서 회의 내용을 음성으로 메모
1. 사용자가 웹 앱을 실행해 로그인
2. '새 메모' 버튼 클릭 → 음성 녹음 시작
3. STT 기술로 실시간 텍스트 변환
4. 메모 저장 시, 자동으로 AI 요약 및 카테고리 분류
5. 전체 메모와 요약본이 웹 화면에 동시 표시됨
6. 데이터는 자체 데이터베이스에 저장되어 웹에서 언제든지 확인 가능

### UI/UX Considerations
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 지원
- **직관적 인터페이스**: 최소한의 클릭으로 메모 작성
- **실시간 피드백**: STT 변환 상태, AI 처리 진행률 표시
- **접근성**: 키보드 네비게이션, 스크린 리더 지원

## Technical Architecture

### System Components

#### Frontend (Next.js)
- **Framework**: Next.js 14 with TypeScript
- **UI Library**: React with Tailwind CSS
- **State Management**: React Context API 또는 Zustand
- **Audio Processing**: Web Audio API, MediaRecorder API

#### Backend (API Routes)
- **Runtime**: Next.js API Routes
- **Database**: PostgreSQL 또는 MySQL
- **ORM**: Prisma
- **Authentication**: NextAuth.js 또는 Clerk

#### External Services
- **STT Service**: Web Speech API 또는 OpenAI Whisper
- **AI Summary**: OpenAI GPT-4 또는 Claude API
- **Category Classification**: OpenAI API 또는 Hugging Face

### Data Models

#### User
```typescript
{
  id: string
  email: string
  name: string
  createdAt: Date
  updatedAt: Date
}
```

#### Memo
```typescript
{
  id: string
  userId: string
  title: string
  content: string
  summary: string
  category: string
  tags: string[]
  audioUrl?: string
  createdAt: Date
  updatedAt: Date
}
```

#### Category
```typescript
{
  id: string
  name: string
  color: string
  userId: string
  createdAt: Date

```

### APIs and Integrations

#### Internal APIs
- `POST /api/memos` - 메모 생성
- `GET /api/memos` - 메모 목록 조회
- `PUT /api/memos/[id]` - 메모 수정
- `DELETE /api/memos/[id]` - 메모 삭제
- `POST /api/memos/[id]/summarize` - AI 요약 생성
- `POST /api/memos/[id]/classify` - 카테고리 분류

#### External APIs
- OpenAI API (GPT-4 for summarization, classification)
- Web Speech API (STT)
- File upload service (audio files)

### Infrastructure Requirements
- **Hosting**: Vercel 또는 AWS
- **Database**: PostgreSQL (Vercel Postgres 또는 AWS RDS)
- **File Storage**: AWS S3 또는 Vercel Blob
- **CDN**: Vercel Edge Network
- **Monitoring**: Vercel Analytics

## Risks and Mitigations

### Technical Challenges

#### STT 정확도 문제
- **Risk**: Web Speech API의 한국어 지원 한계
- **Mitigation**: 
  - OpenAI Whisper API 백업 플랜
  - 사용자 피드백을 통한 정확도 개선
  - 수동 편집 기능 제공

#### AI API 비용 관리
- **Risk**: OpenAI API 사용량 증가로 인한 비용 폭증
- **Mitigation**:
  - 사용량 제한 및 모니터링 시스템
  - 캐싱 전략으로 중복 요청 방지
  - 대안 AI 서비스 검토

#### 실시간 음성 처리 성능
- **Risk**: 브라우저 성능 한계로 인한 지연
- **Mitigation**:
  - Web Workers를 통한 백그라운드 처리
  - 청크 단위 처리로 메모리 최적화
  - 성능 모니터링 및 최적화

### MVP Scope Management

#### 기능 범위 조절
- **Risk**: 너무 많은 기능으로 인한 개발 지연
- **Mitigation**:
  - Phase 1에서 핵심 기능만 우선 구현
  - 사용자 피드백 기반 우선순위 조정
  - 점진적 기능 추가

#### 기술 스택 복잡성
- **Risk**: 새로운 기술 학습으로 인한 지연
- **Mitigation**:
  - 검증된 기술 스택 우선 사용
  - 단계적 기술 도입
  - 충분한 테스트 및 문서화

### Resource Constraints

#### 개발 시간 관리
- **Risk**: 예상보다 긴 개발 시간
- **Mitigation**:
  - 명확한 마일스톤 설정
  - 우선순위 기반 기능 개발
  - MVP 범위 최소화

#### 외부 의존성 관리
- **Risk**: 외부 API 서비스 장애
- **Mitigation**:
  - 다중 백업 서비스 준비
  - 장애 대응 계획 수립
  - 서비스 상태 모니터링

## Appendix

### Research Findings

#### STT 기술 비교
- **Web Speech API**: 무료, 브라우저 내장, 한국어 지원 제한
- **OpenAI Whisper**: 높은 정확도, 유료, 한국어 우수
- **Google Speech-to-Text**: 높은 정확도, 유료, 한국어 우수

#### AI 요약 기술
- **OpenAI GPT-4**: 높은 품질, 유료, 한국어 우수
- **Claude API**: 높은 품질, 유료, 한국어 우수
- **Hugging Face**: 오픈소스, 무료, 한국어 모델 제한

### Technical Specifications

#### 성능 요구사항
- **페이지 로딩 시간**: < 2초
- **STT 응답 시간**: < 1초
- **AI 요약 처리 시간**: < 5초
- **동시 사용자**: 100명 이상

#### 보안 요구사항
- **데이터 암호화**: 전송 및 저장 시 암호화
- **사용자 인증**: JWT 토큰 기반
- **API 보안**: Rate limiting, CORS 설정
- **개인정보 보호**: GDPR 준수

#### 접근성 요구사항
- **WCAG 2.1 AA**: 웹 접근성 가이드라인 준수
- **키보드 네비게이션**: 모든 기능 키보드로 접근 가능
- **스크린 리더**: NVDA, JAWS 등 지원
- **색상 대비**: 최소 4.5:1 비율 유지
